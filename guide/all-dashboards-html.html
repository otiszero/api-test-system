Mỗi layer tự tạo data riêng (self-contained), NHƯNG có shared concern:
→ Integration test tạo User + Product + Order → nếu POST /users bị bug → Integration cũng fail
→ Đây không phải dependency giữa layers, mà là dependency lên API hoạt động đúng
Giải pháp: Nếu Integration fail, check Single API trước để isolate root cause.

💡 RBAC Testing Strategy
Minimum viable (có ngay): TYPE 1 + TYPE 2 + TYPE 4 = ~140 tests. Chỉ cần OpenAPI spec + auth config với 2+ roles + 2 user accounts. AI auto-generate ~85%. Cover 80% RBAC vulnerabilities.

Full coverage (cần Human input): Thêm TYPE 3, 5, 6 = +34 tests. Cần Human viết field permission table + special rules + interaction scenarios. Nhưng đây là nơi bắt được bugs NGUY HIỂM nhất.

Mẹo: Điền permission matrix bằng bảng markdown trong test-rules.md. Mỗi dòng = 1 endpoint, mỗi cột = 1 role, ✅/❌/🔒. AI đọc bảng → generate toàn bộ TYPE 1 + TYPE 2.
