👤
User Layer
QC Engineer interacts via slash commands in Claude Code. Commands orchestrate the entire workflow from setup to reporting.
⚙️
Configuration Layer
QC fills config files (api, auth, db, test-rules) and provides OpenAPI spec. These drive all test generation.
🤖
Generation Layer
AI generates test files based on canonical endpoints. Includes helpers (api-client, auth, db) and 7 test layers.
▶️
Execution Layer
Vitest runs tests with hooks (vitest.setup.ts) that capture evidence for every HTTP request made during tests.
📊
Reporting Layer
Generates markdown reports, bug reports with curl commands, coverage matrix, and interactive HTML dashboard.

  • /generate-* MUST read canonical-endpoints.json first

  • DB layer requires db.config.enabled = true

🛡️ Anti-Hallucination System

❌ The Problem
AI can "hallucinate" endpoints that don't exist based on common patterns:
  • /markets/new-markets → Doesn't exist (actual: /markets/proposed)
  • /markets/popular-markets → Doesn't exist (actual: /markets/favorites)
✅ The Solution
canonical-endpoints.json is the SINGLE SOURCE OF TRUTH for all endpoints.
  1. /analyze runs npm run extract-endpoints
  2. Script parses OpenAPI spec programmatically
  3. Applies endpoint filters (blacklist/whitelist)
  4. Writes generated/canonical-endpoints.json
  5. All /generate-* commands MUST read this file first
  6. Only generate tests for endpoints with isTestable: true
